# vs-arena 블루프린트

## 한 줄 정의
쇼츠에서 물리 배틀 시뮬(무기 볼 배틀)을 "보기만 하던" 글로벌 시청자가, 브라우저에서 직접 매치업을 짜서 돌리고 결과를 링크 한 방으로 공유하는 웹 시뮬레이터.

## 가정
- **UI는 영어**: 타겟이 글로벌 쇼츠 시청자층(저연령 포함)이라 영어 단일 UI로 시작. 한국어는 로드맵.
- **로그인 없음**: 저연령 글로벌 타겟 + 광고 BM이므로 계정/개인정보를 아예 안 받는다.
- **결정론적 시뮬**: 같은 시드 = 같은 전투. 공유 링크가 "같은 경기"를 재현해야 바이럴 루프가 돈다.
- **무기 6종으로 시작**: Pickaxe / Angler(낚싯대) / Sword / Axe / Hammer / Trident. 원본 영상 밈(Pickaxe VS Angler)을 기본 매치업으로 노출.
- **호스팅은 GitHub Pages(무료, 정적)**: 백엔드 없음. 공유는 URL 파라미터로 해결.

## BM 매칭
- **선택: 광고 (콘텐츠 소비형)** — 시청자층은 결제 능력이 낮고(저연령), 가치가 "심심풀이 소비"라 구독 전환이 어렵다. 트래픽 자체가 자산.
- 과금(수익화) 가능 시점: 배포 후 일 방문 500~1,000+ 도달 시 AdSense 부착. 광고 슬롯 자리는 MVP에 미리 잡아둠.
- 2차 수익원(로드맵): 쇼츠 **제작자용** WebM 녹화 내보내기 → 크리에이터 대상 프리미엄(원조 영상 제작자들이 유료 고객이 되는 구조).

## 보안 체크리스트
- **유저 입력(URL 파라미터/시드)**: 무기명은 화이트리스트 매칭, 숫자는 범위 클램프. innerHTML에 유저 값 주입 금지(textContent만) → XSS 차단.
- **공통**: 시크릿 없음(정적 사이트), GitHub Pages HTTPS 기본, 외부 API·키·개인정보 수집 없음.
- 로그인/결제/업로드 없음 → 해당 항목 없음.

## 기술스택
- **바닐라 JS + Canvas 2D** (물리 직접 구현, ~500줄): 물리가 원·선분 충돌 수준이라 엔진 불필요. 번들러 없음 = 빌드 없음 = GitHub Pages에 그대로 배포. 사용자 성향(가벼운 스택, 무료 티어)과 "찍어내기" 목표에 부합.
- 배포: GitHub Pages (무료, HTTPS).

## MVP 범위
**IN**
- 무기 6종 1v1 자동 전투 (회전 무기, 타격 시 무기 성장 = 원본 밈 메커니즘)
- 매치업 빌더: 양측 무기 선택 + HP/속도 슬라이더 + 시드
- 결정론적 리플레이 + 공유 링크 복사 (링크 열면 같은 경기 자동 재생)
- 9:16 세로 아레나, HP바, 승자 연출, 모바일 대응, 광고 슬롯 자리

**OUT (로드맵으로)**
- 토너먼트(4/8강), 커스텀 무기 에디터, WebM 녹화 내보내기, 사운드팩, 리더보드, 다국어, AdSense 실제 부착

## 로드맵
1. GitHub Pages 배포 + 애널리틱스 → 레딧/쇼츠 댓글에 시딩 → 일 500 방문 시 AdSense
2. 토너먼트 모드 + WebM 내보내기 (크리에이터 툴 피벗 검증)
3. 커스텀 무기 에디터 + 프리미엄(무광고/에디터)
